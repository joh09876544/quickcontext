# Support

## Getting Help

Use the repository issue tracker for:

- bug reports
- setup problems
- feature requests
- documentation gaps

## Before Opening An Issue

- Read `README.md`
- Check `AI_DOCS.md` if you are using an AI coding workflow
- Search existing issues first
- Include platform details, error output, and steps to reproduce

## Best Place For Each Question

- Bug report: open an issue
- Feature request: open an issue with the problem and proposed direction
- Contribution question: read `CONTRIBUTING.md`, then open an issue if needed
- Security concern: follow `SECURITY.md`

## What To Include

- operating system
- Python version
- Rust version
- command you ran
- relevant config details without secrets
- full error text
