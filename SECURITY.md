# Security Policy

## Reporting A Vulnerability

Do not post security vulnerabilities in public issues.

If you discover a security issue:

1. Prepare a short report with the affected area, impact, and reproduction steps.
2. Include any proof of concept only if it is necessary to explain the issue.
3. Send the report privately to the maintainers through a direct contact path you control.

If no private contact path is available yet, open an issue only to request a private reporting channel and do not include exploit details.

## Scope

Security issues may include:

- secret exposure
- unsafe file editing behavior
- unsafe IPC behavior
- unintended code execution paths
- dependency-related vulnerabilities

## Handling

Maintainers will review reports, confirm impact, and coordinate a fix before public disclosure when possible.
